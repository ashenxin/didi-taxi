# 后台管理系统：运力与换队审核设计

本文档描述 **运力域**（公司、司机、车辆）与 **换队审核** 的产品与技术共识，与《后台管理系统_运力配置_接口文档.md》配套。  
全站登录与省/市数据域见《后台管理系统_权限清单与鉴权设计.md》及《权限与接口文档》**§4.7**。**计价规则**（`calculate-service` / `fare_rule`）不在本文展开，见《后台管理系统_计价管理_设计.md》与《后台管理系统_计价管理_接口文档.md》。

## 2. 架构边界

- **`capacity-service`**：公司 `Company`、司机 `Driver`、车辆 `Car`、换队申请 `DriverTeamChangeRequest`；HTTP 前缀见接口文档。
- **`admin-api`**：前缀 **`/admin/api/v1/capacity`**（公司及司机、司机关联车辆）、**`/admin/api/v1/capacity/team-change-requests`**（换队）；**Feign → capacity**；**不直连运力库**。
- **前台 `didi-Vue`**：路由与菜单以 `sys_menu` 为准；列表需「看得懂」的名称字段，避免裸 ID。

## 3. 领域模型（与实现对齐）

- **运力公司（Company）**：**一行 = 公司 + 车队（承运单元）**。同一业务公司可在同一省市下有多条记录（共用 `company_no` / `company_name`），以 **车队名称 `team`** 区分；技术主键为 **`id`**，司机归属与换队目标均指向 `company.id`。**车队业务编码 `team_id`**（如 2001、2002）全库唯一，与自增主键 `id` 分离；未在界面填写时由服务端按 `MAX(team_id)+1`（起点 2001）生成，**不回收**已删除行的号码。详见表 `company` 唯一约束 `uk_company_team_id`。
- **司机（Driver）**：归属 **公司**（`companyId`，换队审核期可为空）；**城市编码** `cityCode` 是 **数据域与换队列表筛选** 的关键字段；另存 **`provinceCode` / `provinceName`** 与省市筛选、列表「省/市」展示一致（见《运力配置_接口文档》**§6 库表变更**）。**审核状态快照 `auditStatus`**：**0～3**（待完善 / 审核中 / 通过 / 驳回·需补件），**不再使用值 4**（历史数据可手工将 **`audit_status`/`result_status` 的 4 更新为 3**）。
- **车辆（Car）**：归属一名司机；管理端当前以 **「某司机名下车辆分页」** 暴露为主（一对一场景下通常 0～1 条）。
- **换队申请**：司机从原公司解绑、目标为公司（实现里多为 `toCompanyId`）；**PENDING / APPROVED / REJECTED** 等状态；拒绝需原因。

**与老 TODO 的差异**：领域 REST 以 **`company` / `companyId`** 为准，不单独做「车队」资源列表；库内若有 `team` 等命名以代码为准。

### 3.1 管理端写操作（新增 / 编辑 / 删除）

- **新增**：选省市、填公司编号/名称、车队名称；可选填 **车队业务编码 `teamId`**，留空则系统自动分配。创建前经 BFF **`AdminDataScope#scopeCompanyWrite`** 与登录省/市域对齐（省管/市管不可跨域）。
- **编辑**：仅允许修改 **公司名称 `companyName`** 与 **车队名称 `team`**；**公司编号 `companyNo`、省市、车队业务编码 `team_id` 等不可改**（见接口文档 `PUT`）。
- **删除**：**逻辑删除**（`is_deleted=1`）。若仍存在 **`is_deleted=0` 且 `company_id` 指向本行** 的司机，则拒绝删除。

## 4. 换队审核策略（方案：先解绑）

- 提交申请后：**原归属解除**，**不可接单**（`can_accept_order = 0` 等，以表字段为准）。
- **审核通过**：归属目标公司，恢复可接单。
- **审核拒绝**：不回滚到旧公司；司机仍无归属、仍不可接单，需重新申请。
- **并发**：审核动作须 **CAS**（仅 `PENDING` 可更新），避免重复审核。

接单链路与司机端必须校验：**有有效归属** 且 **允许接单**，与换队状态一致（见各服务实现）。

---

## 5. 数据权限

- 与非订单域相同：**`AdminDataScope`** 裁剪 **公司列表、司机列表、司机车辆、换队列表/待审数/详情/审核**（详见《权限与接口文档》**§4.7**）。
- **换队**：列表按司机 `cityCode`（及省前缀）在 **`capacity-service`** 侧过滤；详情/审核前 BFF 校验 **`driverCityCode`**；跨域 **404**。

---

## 6. 风险与后续

- **一致性**：申请状态与司机 `companyId` / `can_accept_order` 同学务事务更新。
- **可观测性**：审核写建议记操作日志（操作者、资源 id、结果）。
- **隐私**：脱敏、细粒度按钮权限可后置；字段预留 `maskedPhone` 等扩展。
