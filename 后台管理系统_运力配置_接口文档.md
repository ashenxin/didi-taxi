# 后台管理系统：运力与换队接口契约

与《后台管理系统_运力配置_设计.md》配套。前端 **只调 `admin-api`**；表中 **capacity 直连路径** 供联调与 **BFF Feign** 对照。

---

## 1. 通用约定

- **鉴权**：**`Authorization: Bearer <JWT>`**（除全站登录接口外）。
- **响应**：**`ResponseVo<T>`**；管理端与 capacity  successful 业务码多为 **200**（与各自 `ExceptionCode` 一致）。
- **数据域**：见《后台管理系统_权限与接口文档.md》**§4.7**（BFF 合并省/市；换队带 `driverCityCode` 校验）。

---

## 2. `admin-api` 对外接口

**运力视图前缀**：`/admin/api/v1/capacity`

| Method | Path | 说明 |
|--------|------|------|
| GET | `/companies` | 公司分页；Query：`pageNo`、`pageSize`、`provinceCode?`、`cityCode?`、`companyNo?`、`companyName?` |
| GET | `/companies/{id}` | 公司详情（未删除）；BFF 删除前用于域校验 |
| POST | `/companies` | 新增「公司+车队」；Body：`provinceCode`、`provinceName`、`cityCode`、`cityName`、`companyNo`、`companyName`、`team`、`teamId?`（`teamId` 为空则下游自动生成全库唯一业务编码） |
| PUT | `/companies/{id}` | 仅更新 **公司名称、车队名称**；Body：`companyName`、`team`（均必填非空）；BFF 域校验后转发 |
| DELETE | `/companies/{id}` | 逻辑删除；存在归属司机时下游返回业务错误 |
| GET | `/drivers` | 司机分页；Query：`pageNo`、`pageSize`、`companyId?`、`name?`、`phone?`、`online?`、`provinceCode?`、`cityCode?`、`canAcceptOrder?`、`auditStatus?` |
| GET | `/drivers/{driverId}` | **司机档案详情**；成功时 **`data` 为 `AdminDriverDetailVO`**（证件/驾驶证日期/影像 OSS URL/状态等，**不含密码**）；可附带 **`companyName`**（BFF 按 `companyId` 查公司）；先校验司机 `cityCode` 数据域，越界 **404** |
| GET | `/drivers/{driverId}/cars` | 某司机名下车辆分页；先校验司机城市域 |
| GET | `/cars` | **独立车辆分页**；Query：`pageNo`、`pageSize`、`provinceCode?`、`cityCode?`、`companyId?`、`driverId?`、`driverPhone?`、`carNo?`、`brandName?`、`rideTypeId?`（省/市与登录域合并；越界 403） |
| GET | `/cars/{id}` | **车辆详情**（含司机/公司字段）；车辆城市越界 **404** |

**换队前缀**：`/admin/api/v1/capacity/team-change-requests`

| Method | Path | 说明 |
|--------|------|------|
| GET | `/pending-count` | 待审核条数（角标）；带登录省/市域 |
| GET | `/` | 分页；Query：`pageNo`、`pageSize`、`status?`、`driverId?`、`driverPhone?`、`startTime?`、`endTime?`（域由 BFF 注入） |
| GET | `/{id}` | 详情；跨域 **404** |
| POST | `/{id}/approve` | 通过；body 可选 `reviewReason`；Query `reviewedBy?` |
| POST | `/{id}/reject` | 拒绝；body **`reviewReason` 必填**；Query `reviewedBy?` |

**说明**：司机端 **提交** 换队申请若走 app/API，以 capacity -app 路径为准；管理端 BFF **透传提交** 未接时以代码为准。

## 3. `capacity-service` 直连接口（摘要）

**公司业务**

| Method | Path |
|--------|------|
| GET | `/api/v1/companies` |
| GET | `/api/v1/companies/{id}` |
| POST | `/api/v1/companies` |
| PUT | `/api/v1/companies/{id}` | Body：`companyName`、`team`（仅更新这两字段） |
| DELETE | `/api/v1/companies/{id}` |

**司机**

| Method | Path |
|--------|------|
| GET | `/api/v1/drivers` |
| GET | `/api/v1/drivers/{driverId}` |
| GET | `/api/v1/drivers/{driverId}/cars` |
| POST | `/api/v1/drivers/{driverId}/online` |
| GET | `/api/v1/drivers/{driverId}/accept-readiness` |

司机分页 Query 含：`pageNo`、`pageSize`、`companyId`、`name`、`phone`、`online`、**`provinceCode`、`cityCode`**、**`canAcceptOrder`**、**`auditStatus`**。  
- **省筛选**：优先 **`province_code = provinceCode`（国标省码 6 位）**；历史行 `province_code` 为空时退回 **`city_code LIKE 省码前两位%`**（与仅市档案的旧数据兼容）。  
- **市筛选**：`cityCode` 精确等于 `city_code`。  
- **审核状态 `auditStatus`**：取值 **0～3**（0 待完善、1 审核中、2 通过、3 驳回/需补件）；**已不再使用值 4**（历史「暂停接单」若仍落库可执行迁移脚本，见下文「库表变更」）。

**换队（管理）**

| Method | Path |
|--------|------|
| GET | `/api/v1/admin/driver-team-change-requests` |
| GET | `/api/v1/admin/driver-team-change-requests/{id}` |
| POST | `/api/v1/admin/driver-team-change-requests/{id}/approve` |
| POST | `/api/v1/admin/driver-team-change-requests/{id}/reject` |

换队分页可带 **`provinceCode`、`cityCode`**（按司机档案过滤）；列表/详情 VO 含 **`driverCityCode`**（供 BFF 域校验）。

**车辆**

| Method | Path |
|--------|------|
| GET | `/api/v1/cars` |
| GET | `/api/v1/cars/{id}` |

车辆分页 Query 常用：`pageNo`、`pageSize`、`provinceCode?`、`cityCode?`、`companyId?`、`driverId?`、`driverPhone?`、`carNo?`、`brandName?`、`rideTypeId?`。  
其中 `provinceCode`/`cityCode` 按 **车辆 `city_code`** 过滤（省筛选为市码前缀匹配）。

**生产注意**：直连 capacity 若暴露公网，须自带鉴权或与网关策略一致；当前默认 **BFF 鉴权 + 内网下游**。

## 5. 错误码（摘要）

| 场景 | 典型 HTTP / code |
|------|------------------|
| 未登录 | 401 |
| 筛选项越出账号省/市 | 403 |
| 资源不在域内 / 不存在（掩蔽） | 404 |
| 参数校验失败 | 400 |
| 下游错误 body | BFF 透传或统一 502 |

## 6. 库表与脚本（`capacity` 库）

脚本路径相对于仓库 **`capacity/src/main/resources/sql/`**。历史增量迁移脚本已收束，**新环境请直接使用全量 DDL**。

| 脚本 | 说明 |
|------|------|
| **`capacity_schema.sql`** | 全量建库建表（`company` / `driver` / `car` / `driver_team_change_request` / `driver_audit_record`；`driver` 已含 **`province_code`**、**`province_name`** 及 **`idx_driver_province_code`**） |
| **`capacity_seed.sql`** | 全量演示数据（`TRUNCATE` 后写入；与订单/计价联调 id 对齐） |

**`driver.audit_status`（表注释）**：**0** 待完善、**1** 审核中、**2** 通过、**3** 驳回/需补件（**无 4**；若存量库仍存在 **4**，可手工 `UPDATE` 为 **3**）。
