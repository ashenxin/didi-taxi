# 乘客端登录接口文档（MVP）

基于《`乘客端_登录设计.md`》整理，用于前后端联调与自测。  
对外统一经 `gateway` 访问（本地默认 `http://127.0.0.1:8080`），接口前缀均为 `/app/api/v1/auth`。

---

## 0. 通用约定

### 0.1 请求头

- **JSON 请求**：`Content-Type: application/json`
- **鉴权（除白名单外）**：`Authorization: Bearer <accessToken>`

### 0.2 响应结构

统一 `ResponseVo<T>`：

```json
{ "code": 200, "msg": "success", "data": {} }
```

### 0.3 网关白名单（无需 token）

仅以下 3 个接口 **无需** `Authorization`：

- `POST /app/api/v1/auth/login-password`
- `POST /app/api/v1/auth/login-sms`
- `POST /app/api/v1/auth/sms/send`

其余 `/app/**` 均需要 token（无“无登录试玩”需求）。

### 0.4 JWT（Access Token）约定

- **签名**：HMAC（与网关共享密钥）
- **`sub`**：`customerId`（字符串）
- **`aud`**：必须为 `app-bff`（网关开启 `aud` 校验时需要）
- **过期**：7 天（首期仅客户端删除 token 视为退出；不做服务端即时失效）

> 后续业务接口使用乘客身份时，统一从网关注入头 `X-User-Id`（即 JWT `sub`）获取，不信任客户端传 `passengerId`。

## 1. 发送短信验证码

### 1.1 接口

- **POST** `/app/api/v1/auth/sms/send`

### 1.2 请求

```json
{
  "phone": "13800138000"
}
```

字段说明：
- `phone`：手机号，必须为大陆手机号格式（建议 `^1\\d{10}$`）

### 1.3 响应

成功：

```json
{ "code": 200, "msg": "success", "data": null }
```

### 1.4 错误码与文案

- `400`：参数错误（例如手机号格式不正确）
- `429`：频控/上限
  - 发送过于频繁：`发送过于频繁，请稍后再试`
  - 今日发送次数上限（自然日 5 次）：`今日验证码发送次数已达上限，请明天再试`

### 1.5 业务规则（MVP）

- 发送间隔：60s
- 发送日上限：自然日 5 次（按手机号维度）
- OTP 长度：6 位数字
- OTP TTL：300s

## 2. 验证码登录（登录即注册）

### 2.1 接口

- **POST** `/app/api/v1/auth/login-sms`

### 2.2 请求

```json
{
  "phone": "13800138000",
  "code": "123456"
}
```

字段说明：
- `phone`：手机号
- `code`：短信验证码

### 2.3 响应

成功：

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "accessToken": "<jwt>",
    "tokenType": "Bearer",
    "expiresIn": 604800,
    "customer": {
      "id": 10001,
      "phone": "13800138000",
      "nickname": "乘客A"
    }
  }
}
```

字段说明：
- `accessToken`：JWT access token
- `tokenType`：固定 `Bearer`
- `expiresIn`：过期秒数（7 天 = 604800）
- `customer.id`：`customerId`（同时也会写入 JWT `sub`）
- `customer.phone`：手机号（仅用于展示/确认；鉴权不依赖它）
- `customer.nickname`：昵称（可空）

### 2.4 错误码与文案

- `401`：验证码错误或已过期：`验证码错误或已过期`
- `403`：账号冻结：`账号已冻结，请联系客服`
- `429`：登录失败次数过多（自然日 5 次）：`登录失败次数过多，请明天再试`

### 2.5 业务规则（MVP）

- OTP 校验成功后必须 **一次性删除**（防复用）
- 若手机号未注册：自动创建 `customer`（`password_hash` 为空）
- 若账号冻结：禁止登录
- 登录失败计数：密码失败 + 验证码失败 **合并计数**，自然日上限 5 次；达到上限后当日禁止登录成功

## 3. 密码登录

### 3.1 接口

- **POST** `/app/api/v1/auth/login-password`

### 3.2 请求

```json
{
  "phone": "13800138000",
  "password": "plaintext"
}
```

字段说明：
- `phone`：手机号
- `password`：明文密码（仅 HTTPS 传输；服务端存 BCrypt 摘要）

### 3.3 响应

成功：同「验证码登录」成功响应（返回 token + customer）。

### 3.4 错误码与文案

- `401`：
  - 手机号或密码错误：`手机号或密码错误`
  - 该手机号未设置密码（`password_hash` 为空）：`请使用验证码登录`
- `403`：账号冻结：`账号已冻结，请联系客服`
- `429`：登录失败次数过多（自然日 5 次）：`登录失败次数过多，请明天再试`

### 3.5 业务规则（MVP）

- `password_hash` 为空：直接拒绝，引导走验证码登录
- 登录失败计数：与验证码登录合并计算（自然日 5 次）

## 4. 自测用例（示例）

### 4.1 发送验证码

```bash
curl -X POST 'http://127.0.0.1:8080/app/api/v1/auth/sms/send' \
  -H 'Content-Type: application/json' \
  -d '{"phone":"13800138000"}'
```

### 4.2 验证码登录

```bash
curl -X POST 'http://127.0.0.1:8080/app/api/v1/auth/login-sms' \
  -H 'Content-Type: application/json' \
  -d '{"phone":"13800138000","code":"123456"}'
```

### 4.3 密码登录

```bash
curl -X POST 'http://127.0.0.1:8080/app/api/v1/auth/login-password' \
  -H 'Content-Type: application/json' \
  -d '{"phone":"13800138000","password":"plaintext"}'
```

---

## 5. 备注（与后续业务接口的衔接）

- 网关验签通过后会注入 `X-User-Id`（值为 JWT `sub`）。  
- 后续订单/行程/资料等接口，凡涉及乘客身份，统一从 `X-User-Id` 获取 `customerId`，不再让客户端提交 `passengerId`。
