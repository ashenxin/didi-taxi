# 司机端登录/注册接口文档（driver-api 对外）

本文档基于《`司机端_登录注册设计.md`》，给出**可直接联调**的司机端登录/注册接口说明与示例。

- **统一网关入口**：`gateway` `http://127.0.0.1:8080`
- **BFF 服务**：`driver-api`（路由前缀 `/driver/**`）
- **统一响应包装**：`{ code, msg, data }`
- **认证方式**：登录/注册成功后获取 `accessToken`，后续请求：
  - `Authorization: Bearer <accessToken>`
  - 网关会在校验通过后注入 `X-User-Id = <JWT sub>`

**关联文档**：

- 《`乘客司机端_最小闭环接口调用文档.md`》— 总览与乘客端。
- 《`司机端_上线听单与接单设计.md`》— **派单确认、调度、WebSocket、`token_version`、订单字段**等已拍板内容（**推荐优先阅读**）。
- 《`司机端_登录注册设计.md`》— JWT **`token_version` + Redis**、登出强制作废。

实现落地前，**以现有 HTTP 接口为准**；目标形态见上述专项文档。

## 1. 通用约定

### 1.1 JWT 约定（司机端）

- `sub`：司机ID（`driver.id`）
- `aud`：`driver-bff`（必须；网关开启 aud 校验）
- `exp`：**8 小时**自然过期；响应字段 **`expiresIn` = 28800**（秒），与 JWT `exp` 一致
- **`tv`**（JWT claim，与 **`token_version`** 同义）：须与 Redis **`driver:tv:{driverId}`** 当前值一致；**同一司机全局仅一个有效会话**（新登录或登出均会递增版本）；见《`司机端_登录注册设计.md`》**§4.4**。

### 1.2 统一错误码（建议）

- `200`：成功
- `400`：参数非法（手机号格式、验证码格式等）
- `401`：登录/注册失败（验证码错误或过期、手机号或密码错误、未设置密码）
- `403`：禁止接单（用于业务接口，不用于禁止登录）
- `409`：手机号已注册（若注册接口选择“强区分注册/登录”策略）
- `429`：发送过于频繁 / 今日次数已达上限 / 登录失败次数过多（当日封禁）
- `502`：下游服务不可用

## 2. 接口列表（Public：网关需放行）

以下接口均为 `POST`，且建议在网关白名单放行：

- `/driver/api/v1/auth/sms/send`
- `/driver/api/v1/auth/register-sms`
- `/driver/api/v1/auth/register-password`
- `/driver/api/v1/auth/login-sms`
- `/driver/api/v1/auth/login-password`

**需登录（不在 Public 白名单）**：例如 **`POST /driver/api/v1/auth/logout`** — 必须带 `Authorization: Bearer`，见 [§9](#9-退出登录)。

## 3. 发送短信验证码

### 3.1 请求

- **POST** `/driver/api/v1/auth/sms/send`
- **Content-Type**: `application/json`

```json
{ "phone": "13900000000" }
```

### 3.2 成功响应

```json
{ "code": 200, "msg": "success", "data": null }
```

### 3.3 失败响应示例

- `429` 发送过于频繁/日上限：

```json
{ "code": 429, "msg": "发送过于频繁，请稍后再试", "data": null }
```

### 3.4 curl 示例

```bash
curl -X POST "http://127.0.0.1:8080/driver/api/v1/auth/sms/send" \
  -H "Content-Type: application/json" \
  -d '{"phone":"13900000000"}'
```

## 4. 短信注册（OTP 注册）

> 说明：注册成功后返回 token。司机是否可接单由审核状态决定，**不影响登录/注册**。

### 4.1 请求

- **POST** `/driver/api/v1/auth/register-sms`

```json
{ "phone": "13900000000", "code": "123456" }
```

### 4.2 成功响应

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "accessToken": "<jwt>",
    "tokenType": "Bearer",
    "expiresIn": 28800,
    "driver": {
      "id": 80001,
      "phone": "13900000000",
      "auditStatus": 0
    }
  }
}
```

### 4.3 失败响应示例

- `401` 验证码错误/过期：

```json
{ "code": 401, "msg": "验证码错误或已过期", "data": null }
```

- `409` 已注册（若采用强区分策略）：

```json
{ "code": 409, "msg": "该手机号已注册，请直接登录", "data": null }
```

### 4.4 curl 示例

```bash
curl -X POST "http://127.0.0.1:8080/driver/api/v1/auth/register-sms" \
  -H "Content-Type: application/json" \
  -d '{"phone":"13900000000","code":"123456"}'
```

## 9. 退出登录

司机主动登出由 **`driver-api`** 提供；**`capacity`** 内网认证接口不负责 JWT 作废。

### 9.1 请求

- **POST** `/driver/api/v1/auth/logout`
- **Content-Type**：`application/json`（可无 body；实现接受空 body）
- **鉴权（必填）**：`Authorization: Bearer <accessToken>`
  - JWT 须为 **HTTP API 用**（claim **`audit=1`**），与登录/注册返回的 token 一致。
  - **不得**将本接口加入网关或 BFF 的 Public 白名单；未带 token 或 token 无效时应 **401**。

### 9.2 成功响应

```json
{ "code": 200, "msg": "success", "data": null }
```

（与工程统一 `ResponseVo` 风格一致。）

### 9.3 服务端语义（已实现）

- 从合法 JWT 解析 **`sub`（司机 ID）**，对 Redis 中该司机的 **`token_version` 执行递增**（`INCR`）。
- JWT 内携带的 **`tv` claim** 须与 Redis 当前值一致才视为有效；登出后旧 token 的 **`tv` 落后**，后续请求在 **`driver-api`** 的 JWT 校验中被拒绝（**401**，提示登录已失效）。
- **幂等**：多次调用登出均为再次递增版本号；旧 token 在第一次成功后即已不可用。
- **与登录**：任意一次登录成功也会递增 `token_version` 并签发新 JWT，效果上同样是「仅保留一个有效会话」。

**Redis 键（实现）**：`driver:tv:{driverId}`，值为当前版本号（字符串/整数均可，以 `StringRedisTemplate` 递增为准）。无 Redis 的本地测试环境可退化为进程内 Map（仅单机）。

### 9.4 失败响应示例

- **401** 未带 `Authorization`、token 签名/`aud` 无效、**`tv` 与 Redis 不一致**（例如已登出或他处已登录刷新会话）：

```json
{ "code": 401, "msg": "缺少或非法的 Authorization", "data": null }
```

或（BFF 过滤器）：

```json
{ "code": 401, "msg": "登录已失效，请重新登录", "data": null }
```

### 9.5 客户端建议

- 收到 **200** 后删除本地 **`accessToken`** 等凭据。
- 若已建立 **WebSocket**，应主动断开；presence / 听单与派单侧协同见《`司机端_登录注册设计.md`》**§12** 与《`司机端_上线听单与接单设计.md`》。

### 9.6 curl 示例

```bash
curl -X POST "http://127.0.0.1:8080/driver/api/v1/auth/logout" \
  -H "Authorization: Bearer <jwt>" \
  -H "Content-Type: application/json" \
  -d '{}'
```
