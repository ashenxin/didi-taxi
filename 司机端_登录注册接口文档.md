# 司机端登录/注册接口文档（driver-api 对外）

本文档基于《`司机端_登录注册设计.md`》，给出**可直接联调**的司机端登录/注册接口说明与示例。

- **统一网关入口**：`gateway` `http://127.0.0.1:8080`
- **BFF 服务**：`driver-api`（路由前缀 `/driver/**`）
- **统一响应包装**：`{ code, msg, data }`
- **认证方式**：登录/注册成功后获取 `accessToken`，后续请求：
  - `Authorization: Bearer <accessToken>`
  - 网关会在校验通过后注入 `X-User-Id = <JWT sub>`

**关联文档**：

- 《`乘客司机端_最小闭环接口调用文档.md`》— 总览与乘客端。
- 《`司机端_上线听单与接单设计.md`》— **派单确认、调度、WebSocket、`token_version`、订单字段**等已拍板内容（**推荐优先阅读**）。
- 《`司机端_登录注册设计.md`》— JWT **`token_version` + Redis**、登出强制作废。

实现落地前，**以现有 HTTP 接口为准**；目标形态见上述专项文档。

## 1. 通用约定

### 1.1 JWT 约定（司机端）

- `sub`：司机ID（`driver.id`）
- `aud`：`driver-bff`（必须；网关开启 aud 校验）
- `exp`：**8 小时**自然过期；响应字段 **`expiresIn` = 28800**（秒），与 JWT `exp` 一致
- **`token_version`**（目标，已拍板）：与 Redis 中当前版本一致则有效；**同一手机号全局仅一个有效会话**；见《`司机端_登录注册设计.md`》**§4.4**。

### 1.2 统一错误码（建议）

- `200`：成功
- `400`：参数非法（手机号格式、验证码格式等）
- `401`：登录/注册失败（验证码错误或过期、手机号或密码错误、未设置密码）
- `403`：禁止接单（用于业务接口，不用于禁止登录）
- `409`：手机号已注册（若注册接口选择“强区分注册/登录”策略）
- `429`：发送过于频繁 / 今日次数已达上限 / 登录失败次数过多（当日封禁）
- `502`：下游服务不可用

## 2. 接口列表（Public：网关需放行）

以下接口均为 `POST`，且建议在网关白名单放行：

- `/driver/api/v1/auth/sms/send`
- `/driver/api/v1/auth/register-sms`
- `/driver/api/v1/auth/register-password`
- `/driver/api/v1/auth/login-sms`
- `/driver/api/v1/auth/login-password`

## 3. 发送短信验证码

### 3.1 请求

- **POST** `/driver/api/v1/auth/sms/send`
- **Content-Type**: `application/json`

```json
{ "phone": "13900000000" }
```

### 3.2 成功响应

```json
{ "code": 200, "msg": "success", "data": null }
```

### 3.3 失败响应示例

- `429` 发送过于频繁/日上限：

```json
{ "code": 429, "msg": "发送过于频繁，请稍后再试", "data": null }
```

### 3.4 curl 示例

```bash
curl -X POST "http://127.0.0.1:8080/driver/api/v1/auth/sms/send" \
  -H "Content-Type: application/json" \
  -d '{"phone":"13900000000"}'
```

## 4. 短信注册（OTP 注册）

> 说明：注册成功后返回 token。司机是否可接单由审核状态决定，**不影响登录/注册**。

### 4.1 请求

- **POST** `/driver/api/v1/auth/register-sms`

```json
{ "phone": "13900000000", "code": "123456" }
```

### 4.2 成功响应

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "accessToken": "<jwt>",
    "tokenType": "Bearer",
    "expiresIn": 28800,
    "driver": {
      "id": 80001,
      "phone": "13900000000",
      "auditStatus": 0
    }
  }
}
```

### 4.3 失败响应示例

- `401` 验证码错误/过期：

```json
{ "code": 401, "msg": "验证码错误或已过期", "data": null }
```

- `409` 已注册（若采用强区分策略）：

```json
{ "code": 409, "msg": "该手机号已注册，请直接登录", "data": null }
```

### 4.4 curl 示例

```bash
curl -X POST "http://127.0.0.1:8080/driver/api/v1/auth/register-sms" \
  -H "Content-Type: application/json" \
  -d '{"phone":"13900000000","code":"123456"}'
```

## 5. 密码注册（OTP 校验 + 设置密码）

> 说明：密码注册必须绑定短信验证码，避免仅凭手机号“占号”。

### 5.1 请求

### 5.3 失败响应示例

```json
{ "code": 401, "msg": "验证码错误或已过期", "data": null }
```

### 5.4 curl 示例

## 8. 登录后调用示例（需要 Authorization）

以“拉取指派订单”为例（当前工程已有接口）：

- **GET** `/driver/api/v1/orders/assigned?driverId=80001`

```bash
curl "http://127.0.0.1:8080/driver/api/v1/orders/assigned?driverId=80001" \
  -H "Authorization: Bearer <jwt>"
```

> 安全建议（后续实现时落地）：业务接口应以 `X-User-Id`（JWT sub）为准，忽略或强校验请求中的 `driverId`，防止伪造。

## 9. 规划中的能力（未在本文展开协议细节）

### 9.1 退出登录（`POST /driver/api/v1/auth/logout`）

- **目标**：司机主动退出时，**服务端可配合**清除/失效会话侧状态（若引入 refresh/黑名单则另议），并与 **WebSocket 断开、presence 下线** 协同；详见《`司机端_登录注册设计.md`》**§6.6** 与 **§12**。
- **网关**：若 `logout` 需带 JWT，则**不应**放入 Public 白名单；实现时明确是否允许未登录调用（通常否）。

### 9.2 WebSocket + presence

- **目标**：司机端建立 **WebSocket** 长连接，维护 **presence**（在线/可派单/可确认），并接收 **待确认派单**（含 **`offerExpiresAt`**，超时仅服务端裁决）。
- **鉴权**：握手携带 Token 或短期票据（实现时确定）；与 HTTP **同一 JWT 体系**。
- **详细语义**：见《`乘客司机端_最小闭环接口调用文档.md`》**§4.5、§6.2**；URL、子协议、消息格式 **待开发阶段补充**。
