# 后台管理系统：计价管理接口契约

与《后台管理系统_计价管理_设计.md》配套。前端 **管理端只调 `admin-api`**；表中 **calculate 直连路径** 供联调与 **BFF Feign** 对照。

---

## 1. 通用约定

- **鉴权**：**`Authorization: Bearer <JWT>`**（除全站登录接口外）。
- **响应**：**`ResponseVo<T>`**； successful 业务码多为 **200**。
- **数据域**：见《后台管理系统_权限与接口文档.md》**§4.7**（列表省、市与登录域合并；读写 body 省、市锁定；跨域 **404**）。

---

## 2. `admin-api` 对外接口（计价规则）

**前缀**：`/admin/api/v1/pricing/fare-rules`

| Method | Path | 说明 |
|--------|------|------|
| GET | `/` | 计价规则分页。Query：`pageNo`（默认 1）、`pageSize`（默认 10）、**`companyId?`**、`provinceCode?`、`cityCode?`、`productCode?`、`ruleName?`（模糊）、**`active?`**（`1` 生效中：`effective_to` 为空或大于当前时间；`0` 已失效：`effective_to` 非空且小于等于当前时间） |
| GET | `/{id}` | 详情；**`data` 为 `AdminFareRuleVO`**（含 **`companyId`、`companyNo`、`companyName`（BFF 填充）** 及规则字段）；不在域内 **404** |
| POST | `/` | 新建；Body：**`FareRuleUpsertBody`**（见 §5）；返回 **`data` 为新建规则 `id`（Long）** |
| PUT | `/{id}` | 更新；Body 同创建；成功 **`data` 常为 null** |
| DELETE | `/{id}` | 逻辑删除 |

**`POST` / `PUT` Body 字段（`FareRuleUpsertBody`）**  

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `companyId` | Long | 是 | 运力公司主键，对应 `capacity.company.id` |
| `companyNo` | String | 否 | 可省略；保存时 BFF 按公司详情 **覆盖** 为 `company_no` |
| `provinceCode` | String | 是 | 国标省码等，须与公司档案一致 |
| `cityCode` | String | 是 | 市/区县码，须与公司档案一致 |
| `productCode` | String | 是 | 产品线编码，与订单约定一致（如 ECONOMY） |
| `ruleName` | String | 否 | 展示名 |
| `effectiveFrom` | LocalDateTime | 是 | 生效开始 |
| `effectiveTo` | LocalDateTime | 否 | 结束时间；**空表示长期有效** |
| `baseFare`～`maximumFare` | 金额类 | 见校验 | 起步价、含里程/时长、单价、最低/封顶（详见 calculate DTO） |

## 3. `calculate-service` 直连接口（摘要）

### 3.1 计价规则

**前缀**：`/api/v1/fare-rules`

| Method | Path | 说明 |
|--------|------|------|
| GET | `/` | 分页；Query 与 BFF 转发参数一致（含 `companyId`、`active` 等） |
| GET | `/{id}` | 详情；未删除 |
| POST | `/` | 创建；Body：`FareRuleUpsertBody`（含 **`companyId`、`companyNo`**） |
| PUT | `/{id}` | 更新 |
| DELETE | `/{id}` | 逻辑删除 |

**业务校验（服务端）**：同一 **`company_id` + province + city + product_code`** 下，`is_deleted = 0` 的规则 **`effective` 区间不得重叠**；否则返回 **业务错误**（HTTP 通常仍为 200，`code` 非 200，以全局异常处理器为准）。

### 3.2 费用预估（非管理菜单，供乘客链路）

| Method | Path | 说明 |
|--------|------|------|
| POST | `/api/v1/calculate/estimate` | Body：`companyId`、`provinceCode`、`cityCode`、`productCode`、`distanceMeters`、`durationSeconds`；未匹配规则时 **404** |

## 4. `admin-api` Feign 对照

**`CalculateClient`**（`services.calculate.base-url`）：`page`、`detail`、`create`、`update`、`delete`，路径对应 §3.1。

## 5. VO 与错误码（摘要）

- **列表项 / 详情 `AdminFareRuleVO`**：`id`、`companyId`、`companyNo`、`companyName`、`provinceCode`、`cityCode`、`productCode`、`ruleName`、`effectiveFrom`、`effectiveTo`、计费明细字段、`createdAt`、`updatedAt`。

| 场景 | 典型 HTTP / code |
|------|------------------|
| 未登录 | 401 |
| 筛选项越出账号省/市 | 403 |
| 规则 / 公司不在域内 | 404 |
| 参数校验失败（含公司与省市不一致、区间重叠） | 400 或业务码（见实现） |
| 下游错误 | BFF 透传或统一 502 |

## 6. 库表与脚本（`calculate` 库）

脚本路径相对于仓库 **`calculate/src/main/resources/sql/`**。

| 脚本 | 说明 |
|------|------|
| **`calculate_schema.sql`** | 全量建库建表（**`fare_rule`**，含 **`company_id`、`company_no`** 及索引 **`idx_fare_rule_company_scope`**） |
| **`calculate_seed.sql`** | 全量种子：演示计价规则 + estimate 兜底幂等插入（公司 id 与 **`capacity_seed.sql`** 对齐） |

测试用 **H2**：`calculate/src/test/resources/schema-test.sql` 中 **`fare_rule`** 定义需与线上一致。

## 7. 前端路由（`didi-Vue` 约定）

| 路由 | 视图 | 说明 |
|------|------|------|
| `/pricing/fare-rules` | `pricing/FareRuleListView.vue` | 列表 + 公司筛选 |
| `/pricing/fare-rules/new` | `pricing/FareRuleEditView.vue` | 新建（公司下拉，选公司带出省/市） |
| `/pricing/fare-rules/:id` | `pricing/FareRuleEditView.vue` | 编辑 |

菜单与权限标识以 **`sys_menu.perms`** 为准（含 **`visible=0`** 的按钮行）；**无独立 `sys_permission` 表**，与《后台管理系统_权限与接口文档.md》**§2** 一致（如 `pricing:fare-rule:list`）。
