# 司机端登录/注册 PRD（产品需求）

本文档描述司机端（司机 App/H5）登录与注册的产品目标、流程与体验口径。  
接口契约与技术实现分别见《司机端_登录注册_API.md》《司机端_登录注册_TECH.md》。

---

## 1. 背景与目标

- 支持司机端 **注册** 与 **登录**：
  - 短信注册、密码注册
  - 短信登录、密码登录
- 登录/注册成功后返回 Access Token（JWT），后续请求统一 `Authorization: Bearer <token>`。
- **能否登录 App 与能否接单解耦**：
  - 司机即使审核未通过/被限制接单，也允许登录，用于补件/申诉/查看原因
  - 接单/听单等动作必须严格校验接单资格（见 TECH）

---

## 2. 登录/注册路径

### 2.1 短信注册

- 发送验证码 → 输入验证码 → 注册成功并登录（返回 token）

### 2.2 密码注册

- 发送验证码 → 输入验证码 + 设置密码 → 注册成功并登录（返回 token）

> 说明：密码注册必须绑定短信验证码，避免仅凭手机号“占号”。

### 2.3 短信登录

- 输入验证码 → 登录成功（返回 token）
- 若手机号未注册：提示先注册（不隐式创建司机）

### 2.4 密码登录

- 输入手机号+密码 → 登录成功（返回 token）
- 若未设置密码：提示使用验证码登录或先设置密码

### 2.5 退出登录

- 主动退出后，本机 token 立即失效（同一司机只保留一个有效会话）
- 客户端需清理 token，并断开 WebSocket（若已建立）

---

## 3. 关键规则（MVP）

### 3.1 短信验证码（OTP）

- 6 位数字
- TTL：300 秒
- 校验成功后一次性删除（防复用）

### 3.2 短信发送频控

- 发送间隔：60 秒
- 日上限：5 次/手机号/自然日

### 3.3 登录失败限制（同手机号）

- 密码失败 + 验证码失败合并计数
- 自然日失败上限：5 次
- 达到上限后：当日禁止登录成功（429）

---

## 4. 文案口径（建议首期统一）

- 验证码错误或已过期：`验证码错误或已过期`
- 手机号或密码错误：`手机号或密码错误`
- 未设置密码：`未设置密码，请使用验证码登录或先设置密码`
- 发送过于频繁：`发送过于频繁，请稍后再试`
- 今日验证码发送次数已达上限：`今日验证码发送次数已达上限，请明天再试`
- 登录失败次数过多：`登录失败次数过多，请明天再试`
- 未注册：`该手机号未注册，请先注册`

---

## 5. 验收用例（最小集）

- 发码：60s 频控、日上限
- 短信注册：正确 OTP 注册成功；OTP 不可复用；已注册提示合理
- 密码注册：正确 OTP + 密码注册成功；错误 OTP 失败
- 短信登录：未注册提示先注册；正确 OTP 登录成功
- 密码登录：正确密码成功；未设密提示；错误密码失败
- 登出：登出后旧 token 立即失效；再次登录旧 token 失效（单会话）；服务端对名下 **`ASSIGNED` / `PENDING_DRIVER_CONFIRM`** 订单批量 **`reject`（原因 `DRIVER_LOGOUT`）** 以释放待接指派，并 **下线听单**（与接口文档 §7 一致）。与乘客端「到达前登出代 **取消**」是否完全同口径，以《第一期 MVP…PRD》§5.6 与 gap 总览为准。
