# 后台管理系统：订单管理接口契约

与《后台管理系统_订单管理_设计.md》配套，约定 **`admin-api` 对外路径** 及 **下游 HTTP 形状**（联调以仓库 `OrderClient` / `PassengerClient` / Controller 为准）。

---

## 1. 通用约定

- **鉴权**：除登录外，须 **`Authorization: Bearer <JWT>`**。
- **响应**：**`ResponseVo<T>`**；**`code == 200`** 成功；HTTP 与 `code` 对齐。
- **数据域**：省/市裁剪见《后台管理系统_权限与接口文档.md》**§4.7**。

---

## 2. `admin-api` 对外接口

**前缀**：`/admin/api/v1/orders`

### 2.1 分页列表

| 项 | 说明 |
|----|------|
| **GET** | `/admin/api/v1/orders` |

**Query**

| 参数 | 必填 | 说明 |
|------|------|------|
| `orderNo` | 否 | 订单号 |
| `phone` | 否 | 乘客手机；**无 `orderNo` 时**才参与查询 |
| `provinceCode` | 否 | 与登录域合并 |
| `cityCode` | 否 | 与登录域合并 |
| `status` | 否 | 订单状态 |
| `createdAtStart` | 否 | `yyyy-MM-dd HH:mm:ss` |
| `createdAtEnd` | 否 | `yyyy-MM-dd HH:mm:ss` |
| `pageNo` | 否，默认 1 | |
| `pageSize` | 否，默认 10 | |

**行为摘要**：若需按手机查，先 `GET /api/v1/customers/by-phone` 得 `passengerId`，无用户则返回空页；列表行不返回乘客手机号（见设计文档）。

### 2.2 详情（含事件）

| 项 | 说明 |
|----|------|
| **GET** | `/admin/api/v1/orders/{orderNo}` |

**成功 `data`**：`AdminOrderDetailVO`（订单字段 + `events` 列表 + 详情内乘客手机等，以实现为准）。

**跨域或无订单**：**404**，`msg` 如「订单不存在」。

---

## 3. 下游依赖（`admin-api` Feign）

### 3.1 `order-service`

| Method | Path | 用途 |
|--------|------|------|
| GET | `/api/v1/orders` | 分页；Query 含 `pageNo`、`pageSize` 及订单筛选项（与 BFF 转发一致） |
| GET | `/api/v1/orders/{orderNo}` | 详情 |
| GET | `/api/v1/orders/{orderNo}/events` | 事件列表（建议持久化/查询按 `occurredAt` 有序） |

### 3.2 `passenger-service`

| Method | Path | 用途 |
|--------|------|------|
| GET | `/api/v1/customers/by-phone?phone=` | 列表按手机筛选 |
| GET | `/api/v1/customers/{passengerId}` | 详情补手机号（`passengerId` 路径类型以实现为准） |

**安全**：生产环境建议内网 + 与权限文档一致的内部演进方案。

## 4. 错误码（摘要）

| 场景 | HTTP / code |
|------|-------------|
| 未登录 / token 无效 | 401 |
| 数据域越权（筛选项） | 403 |
| 订单不存在或不在域内 | 404 |
| 下游不可用 | 502 |
| 超时 / 连接失败 | 504 |

## 5. 联调假数据（`order` 库）

| 项 | 说明 |
|----|------|
| **脚本** | `order/src/main/resources/sql/order_seed.sql`（建表见 `order_schema.sql`） |
| **依赖** | 建议已执行 `passenger_seed.sql`（乘客 `10001`～`10003`）及 **`capacity_seed.sql`**（详见《后台管理系统_订单管理_设计.md》**§6**） |
| **订单号** | **`OD202604150001`～`OD202604150015`** 共 15 条；重复执行会先删除同前缀订单及事件再插入 |
| **状态分布** | 含已创建、已分配、已接单、已到达、行程中、已完单、已取消（乘客取消 / 系统类取消文案）等，便于筛选项与详情时间线验收 |
| **事件类型** | 与线上一致：`ORDER_CREATED`、`ORDER_ASSIGNED`、`ORDER_ACCEPTED`、`ORDER_DRIVER_ARRIVED`、`ORDER_TRIP_STARTED`、`ORDER_FINISHED`、`ORDER_CANCELLED` |
| **操作方 `operator_type`** | 与订单服务实现一致：**0 系统、1 乘客、2 司机**（以代码为准，勿与早期表注释混淆） |

**下游路径（订单服务）**：分页 **`GET /api/v1/orders`**（Query 见上文 BFF 转发）、详情 **`GET /api/v1/orders/{orderNo}`**、事件 **`GET /api/v1/orders/{orderNo}/events`**（按 `occurredAt` 升序）。

**前端省市区划**：列表/详情「省/市」文案使用 **`formatGbRegionDisplayName(provinceCode, cityCode)`**，直辖市下 **`city_code` 为区县码** 时展示「省/区县」；区划编码列对直辖市展示 **`310000 / 310104`** 形式（见 `didi-Vue` `utils/regionCodes.js`）。
