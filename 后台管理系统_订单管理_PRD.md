# 后台管理系统：订单管理 PRD（产品需求）

本文档描述管理端「订单管理」的产品目标、页面体验、权限与验收口径。  
接口契约与技术实现分别见《后台管理系统_订单管理_API.md》《后台管理系统_订单管理_TECH.md》。

---

## 1. 背景与目标

- **目标**：为运营/管理人员提供订单查询与排障入口，支持按关键条件检索订单，并查看订单详情与事件时间线。
- **首期范围**：订单列表、订单详情（含事件时间线）、按省/市数据域控制可见范围。
- **非首期**：人工改单、退款对账、导出、隐私脱敏/审计（仅预留扩展口径）。

---

## 2. 角色与权限

- **页面进入权限**：菜单/角色控制「是否能进入订单页」。
- **数据权限**：列表与详情必须遵守 `AdminDataScope`（见《后台管理系统_权限与接口文档.md》§4.7）。
  - **非 SUPER**：查询条件中的 `provinceCode`/`cityCode` 与 JWT 域合并；越权返回 403。
  - **非 SUPER**：详情订单若不在本账号省/市域内，返回 404（统一文案「订单不存在」，不暴露它域信息）。
  - **SUPER**：可按请求条件筛选，不做省/市锁定。

---

## 3. 页面与交互

### 3.1 订单列表

**筛选项**

- 订单号 `orderNo`
- 乘客手机号 `phone`
- 省 `provinceCode`、市 `cityCode`
- 订单状态 `status`
- 创建时间起止 `createdAtStart` / `createdAtEnd`
- 分页 `pageNo` / `pageSize`

**筛选规则**

- **`orderNo` 与 `phone` 同时出现**：以 `orderNo` 为主；仅在无 `orderNo` 时按 `phone` 生效。
- **按手机号筛选且乘客不存在**：返回空列表（业务 200），不报错。

**列表展示**

- **不展示乘客手机号**：首期避免为每行补手机号导致 N+1 与性能风险；如后续必须展示，需引入乘客批量查询或订单快照字段（后置）。
- **省/市文案展示（国标码）**：
  - 直辖市订单若 `city_code` 为区县码（例如上海 `310104`），列表「省/市」列展示「直辖市名/区县名」（例如「上海市/徐汇区」），避免与旧逻辑混淆。
  - 具体前端格式化函数以实现为准（见 API 文档联调说明）。

### 3.2 订单详情

**展示内容**

- 订单主信息（以接口返回为准）
- 事件时间线（按 `occurredAt` 升序展示）
- 乘客手机号：仅在详情内展示（通过 `passengerId` 补充）

**异常与提示**

- 订单不存在或无权限：展示「订单不存在」（对应接口 404）。

**后续增强（非硬性首期）**

- 排障增强：更多时间节点、取消原因/取消方、事件 payload 折叠展示等。

---

## 4. 错误与体验口径

- **状态对齐**：HTTP 状态码与 `ResponseVo.code` 对齐（401/403/404/502/504）。
- **前端提示**：`code != 200` 时统一弹 `msg`（不做细分）。

---

## 5. 联调与验收建议

### 5.1 联调数据准备（可选）

用于演示列表筛选、详情时间线与多状态覆盖。

- `passenger/src/main/resources/sql/passenger_seed.sql`：演示乘客 `10001～10003`
- `capacity/src/main/resources/sql/capacity_seed.sql`：司机/车辆/公司
- `order/src/main/resources/sql/order_seed.sql`：15 笔演示订单（订单号前缀 `OD20260415`）

建议执行顺序：先 `passenger_seed.sql`、`capacity_seed.sql`，再 `order_seed.sql`。

### 5.2 验收清单（首期）

- 列表：按 `orderNo`、按 `phone`（含不存在乘客）、按省市域与时间范围筛选
- 权限：非 SUPER 越权筛选 403；越域详情 404（文案一致）
- 详情：事件时间线按时间顺序展示；详情内展示乘客手机号
