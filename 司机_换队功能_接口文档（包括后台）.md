# 司机端与后台：换队接口文档（driver-api + admin-api + capacity）

本文档给出“司机换队”闭环的 **完整接口契约** 与联调要点，覆盖：

- 司机端（对外）`driver-api`
- 管理后台（对外）`admin-api`
- 下游运力服务（对内）`capacity-service`

并复述管理端“列表/详情/审核”既有内容（取代零散说明），同时补齐司机端“提交/查询/撤销恢复”。

> 统一响应包装：`ResponseVo<T>`（`{ code, msg, data }`）  
> 网关入口示例：`http://127.0.0.1:8080`

---

## 1. 鉴权与一致性约束（司机端）

### 1.1 认证头

- `Authorization: Bearer <accessToken>`
- 网关鉴权通过后注入：`X-User-Id: <driverId>`（字符串形式，等于 JWT `sub`）

### 1.2 driverId 一致性

- 路径参数、Query、JSON body 中若出现 `driverId`，必须与 `X-User-Id` 一致，否则 403：
  - `code=403, msg="禁止操作其他司机数据"`

> 实现建议：司机端接口尽量 **不再让前端传 driverId**，统一以 `X-User-Id` 为准。

---

## 2. 数据模型（核心字段）

### 2.1 换队申请单（`driver_team_change_request`）

- `id`
- `driver_id`
- `from_company_id`
- `to_company_id`
- `status`：`PENDING / APPROVED / REJECTED / CANCELLED`
- `request_reason`
- `requested_by`（由服务端填充）
- `requested_at`
- `reviewed_by` / `reviewed_at` / `review_reason`

### 2.2 司机接单关键字段（`driver`）

- `company_id`：归属运力主体（可为空）
- `can_accept_order`：0/1
- `monitor_status`：0 未听单 / 1 听单中 / 2 服务中

> capacity 侧上线/接单资格校验会要求 `can_accept_order=1`。

---

## 3. 司机端接口（对外：driver-api）

统一前缀：`/driver/api/v1`

### 3.1 搜索目标运力主体（公司/车队）

**GET** `/capacity/companies/search`

Query：

- `cityCode?`：目标城市（可选）
- `companyKeyword?`：公司关键字（可选，模糊匹配）
- `teamKeyword?`：车队关键字（可选，模糊匹配）
- `pageNo`（默认 1）
- `pageSize`（默认 10，建议上限 20）

成功响应 `data`（建议结构）：

```json
{
  "list": [
    {
      "companyId": 800001,
      "companyName": "合作品牌",
      "team": "上海一队",
      "teamId": 2008,
      "cityCode": "310000",
      "cityName": "上海市",
      "provinceCode": "310000",
      "provinceName": "上海市"
    }
  ],
  "total": 1,
  "pageNo": 1,
  "pageSize": 10
}
```

错误码：

- `401` 未登录
- `400` 参数非法

### 3.2 查询“我的换队申请”（当前/最新）

**GET** `/team-change-requests/current`

语义：

- 若存在 `PENDING`，返回该条
- 否则返回最新一条（`APPROVED/REJECTED/CANCELLED`）
- 若没有任何申请，返回 `data=null`

成功响应 `data`（建议结构）：

```json
{
  "id": 123,
  "status": "REJECTED",
  "requestedAt": "2026-04-23 10:00:00",
  "requestReason": "想换到上海",
  "fromCompanyId": 800010,
  "fromTeamName": "杭州二队",
  "toCompanyId": 800020,
  "toTeamName": "上海一队",
  "reviewedAt": "2026-04-23 12:00:00",
  "reviewedBy": "admin",
  "reviewReason": "资料不全"
}
```

### 3.3 提交换队申请

**POST** `/team-change-requests`

Body：

```json
{
  "toCompanyId": 800020,
  "requestReason": "想换到上海"
}
```

服务端行为（必须）：

- `driverId` 从 `X-User-Id` 获取
- `requestedBy` 由服务端填充（建议填 `driverId` 或手机号）
- 一致性校验：
  - 若 `driver.city_code != null`，必须 `driver.city_code == fromCompany.city_code`
- 成功后副作用（先解绑策略）：
  - 写入 `driver_team_change_request` 为 `PENDING`
  - `driver.company_id = null`
  - `driver.can_accept_order = 0`

成功响应 `data`（建议）：

```json
{ "requestId": 123 }
```

常见失败：

- `409`：已有待审核申请（`PENDING`）
- `400`：目标运力不存在 / 参数非法

### 3.4 撤销/放弃换队并恢复接单（方案 A）

**POST** `/team-change-requests/{id}/cancel`

语义：

- 允许在 `PENDING` 或 `REJECTED` 阶段调用
- 调用成功后：
  - `request.status = CANCELLED`
  - `driver.company_id = request.from_company_id`
  - `driver.can_accept_order = 1`
  - `driver.monitor_status = 0`（必须手动重新上线）

失败与幂等：

- 若申请单不存在或不属于当前司机：`404/403`
- 若申请单已 `APPROVED`：`409`（不可撤销）
- 重复撤销：建议返回 `200`（幂等）或返回明确业务提示

## 4. 管理后台接口（对外：admin-api）

> 以下内容对齐《后台管理系统_运力配置_接口文档.md》并做结构化重述。

统一前缀：`/admin/api/v1/capacity/team-change-requests`

### 4.1 待审核数量

**GET** `/pending-count`

说明：

- BFF 注入登录账号的省/市域
- 返回待审条数（用于菜单角标）

### 4.2 分页列表

**GET** `/`

Query：

- `pageNo`（默认 1）
- `pageSize`（默认 10）
- `status?`（默认 `PENDING`）
- `driverId?`
- `driverPhone?`（模糊）
- `startTime?` / `endTime?`（`yyyy-MM-dd HH:mm:ss`）

数据域：

- BFF 注入 `provinceCode/cityCode`（实现以 `AdminDataScope` 为准）

### 4.3 详情

**GET** `/{id}`

越权语义：

- 不在数据域内：返回 `404`（资源掩蔽）

### 4.4 审核通过

**POST** `/{id}/approve`

Body（可选）：

```json
{ "reviewReason": "通过备注（可选）" }
```

Query：

- `reviewedBy?`（当前实现允许占位，建议后续由鉴权注入）

语义：

- 仅 `PENDING` 可审核
- 通过后会把司机归属设为目标运力，并 `can_accept_order=1`

### 4.5 审核拒绝

**POST** `/{id}/reject`

Body（必填）：

```json
{ "reviewReason": "拒绝原因（必填）" }
```

语义：

- 仅 `PENDING` 可审核
- 拒绝后司机保持“解绑且不可接单”（由司机端走“恢复接单”接口闭环）

建议不直接暴露给前端，而是由 `driver-api` 聚合后调用 `capacity` 内部真实提交/撤销接口。

## 6. 联调建议与常见坑

- **提交后立即不可接单**：前端必须强提醒；提交成功后“上线听单”按钮应禁用，并引导查看申请状态。
- **撤销/恢复后需要手动上线**：恢复成功后明确提示“请重新上线听单”。
- **并发冲突**：在“管理端审核”与“司机撤销”同时发生时，必须依赖 CAS 保证仅一个动作成功，另一个返回“已审核/已撤销”等明确提示。
- **目标选择只提交 ID**：名称仅用于搜索与展示，提交必须使用 `toCompanyId`。
