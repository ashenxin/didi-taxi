# 后台管理系统：权限相关表结构与接口契约

本文档与《后台管理系统_权限清单与鉴权设计.md》配套，约定 **数据库表（DDL 落库脚本）** 与 **HTTP 接口**（`admin-api` 对前端、`passenger` 对内供 BFF Feign）。  
**实现代码可后补**；联调以本文与 DDL 为准。

**订单、运力子域**的专用契约另见：《后台管理系统_订单管理_接口文档.md》《后台管理系统_运力配置_接口文档.md》（设计及边界见同名 `_设计.md`）。

---

## 1. DDL 脚本位置

| 文件 | 说明 |
|------|------|
| `passenger/src/main/resources/sql/sys_admin_rbac.sql` | 建表 + 三角色初始化 INSERT；超管用户与「超管挂全菜单」为注释模板，**须手动执行** |

执行前确认连接 **`passenger` 库**（与 `customer` 同库不同表）。

## 2. 表结构一览

| 表名 | 用途 |
|------|------|
| `sys_role` | 角色定义（`code`：SUPER / PROVINCE_ADMIN / CITY_OPERATOR） |
| `sys_user` | 后台用户；`province_code`/`city_code` 表达数据范围；`token_version` 用于踢下线；`username_active` 生成列实现软删后用户名复用 |
| `sys_user_role` | 用户-角色多对多 |
| `sys_menu` | 菜单树；`path`、`component`、`perms` 供前端路由与按钮权限 |
| `sys_role_menu` | 角色可见菜单 = **权限清单** |

**约束（在 `passenger` 服务代码中校验，非 DB CHECK）**：

- **SUPER**：`province_code`、`city_code` 均为 NULL。  
- **PROVINCE_ADMIN**：`province_code` 非 NULL，`city_code` NULL。  
- **CITY_OPERATOR**：二者均非 NULL，且市隶属于省（业务校验）。

---

## 3. 统一响应约定

### 3.1 `admin-api`（对 `didi-Vue`）

与现有模块一致：**`ResponseVo<T>`**，`code == 200` 表示业务成功；HTTP 状态与 `code` 对齐（如 401/403/502）。

### 3.2 `passenger`（对内）

建议与 `passenger` 现有接口一致：**`ResponseVo<T>`**（`code`/`msg`/`data`），便于 Feign 反序列化。

---

## 4. `admin-api` 对外接口（前缀 `/admin/api/v1`）

以下接口除 **登录** 外，均需 **`Authorization: Bearer <JWT>`**。

### 4.1 登录

| 项 | 说明 |
|----|------|
| **Method / Path** | `POST /admin/api/v1/auth/login` |
| **鉴权** | 无 |
| **Content-Type** | `application/json` |

**请求体**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `username` | string | 是 | 登录名 |
| `password` | string | 是 | 明文密码（HTTPS 传输） |

**成功响应 `data`**

| 字段 | 类型 | 说明 |
|------|------|------|
| `accessToken` | string | JWT |
| `tokenType` | string | 固定 `Bearer` |
| `expiresIn` | number | 过期秒数（与 JWT `exp` 一致） |
| `user` | object | 摘要，见下 |

`user` 建议字段：`id`，`username`，`displayName`，`roleCodes`（数组），`provinceCode`，`cityCode`（可 null）。

**失败**：HTTP **401** 或业务码非 200；`msg` 统一 **「用户名或密码错误」**（不区分用户不存在与密码错）。

**实现要点**：`admin-api` Feign 调 `passenger` 校验凭据并取 `token_version`，本地用 `jwt.secret`（环境变量）签发 JWT；claims 含 `sub`=`userId`，`tv`=`token_version`，`exp` 等。

---

### 4.2 当前登录用户（含缓存友好字段）

| 项 | 说明 |
|----|------|
| **Method / Path** | `GET /admin/api/v1/auth/me` |
| **鉴权** | Bearer JWT |

**成功响应 `data`**：与登录返回的 `user` 同级或扩展，至少包含：`id`，`username`，`displayName`，`roleCodes`，`provinceCode`，`cityCode`，`tokenVersion`（与 JWT 内 `tv` 对齐，便于前端展示/调试）。

**失败**：401（token 无效/过期/`tv` 不一致）。

---

### 4.3 当前用户菜单树（权限清单）

| 项 | 说明 |
|----|------|
| **Method / Path** | `GET /admin/api/v1/auth/menus` |
| **鉴权** | Bearer JWT |

**成功响应 `data`**：`MenuNode[]` 树形结构。

**`MenuNode` 建议字段**

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | number | 菜单 id |
| `parentId` | number \| null | 父 id |
| `path` | string | 路由 |
| `name` | string | 标题 |
| `icon` | string \| null | 图标 |
| `component` | string \| null | 前端组件 |
| `perms` | string \| null | 权限串 |
| `sort` | number | 排序 |
| `visible` | boolean | 是否显示 |
| `children` | array | 子节点 |

**说明**：数据来自 `passenger` 按当前用户角色聚合 `sys_menu` + `sys_role_menu`；**SUPER** 等价于全部有效菜单。可配合 Redis 缓存，变更权限时删缓存。

---

### 4.4 修改当前用户密码（可选）

| 项 | 说明 |
|----|------|
| **Method / Path** | `PUT /admin/api/v1/auth/password` |
| **鉴权** | Bearer JWT |

**请求体**：`oldPassword`，`newPassword`。  
**成功后**：`passenger` 侧更新 `password_hash` 且 **`token_version` + 1**；当前 token 失效，客户端需重新登录。

---

### 4.5 操作员管理（省管员：本市操作员 CRUD）

仅 **PROVINCE_ADMIN**（及 SUPER）可访问；**省管员**只能操作 **`province_code` 等于本人所辖省** 下的 **CITY_OPERATOR** 用户。

| Method | Path | 说明 |
|--------|------|------|
| `GET` | `/admin/api/v1/system/operators` | 分页列表；Query：`pageNo`，`pageSize`，`cityCode`（可选），`username`（可选模糊） |
| `POST` | `/admin/api/v1/system/operators` | 创建市操作员 |
| `GET` | `/admin/api/v1/system/operators/{id}` | 详情 |
| `PUT` | `/admin/api/v1/system/operators/{id}` | 更新（显示名、状态、重置密码等，按需） |
| `DELETE` | `/admin/api/v1/system/operators/{id}` | **软删** `sys_user` |

**创建/更新请求体（示例字段）**

| 字段 | 说明 |
|------|------|
| `username` | 登录名 |
| `password` | 明文密码（仅创建或重置时） |
| `displayName` | 展示名 |
| `provinceCode` | 须与当前省管省份一致 |
| `cityCode` | 目标城市，须隶属于该省 |
| `roleCode` | 固定或默认 `CITY_OPERATOR` |

**权限错误**：HTTP **403**。

**实现**：`admin-api` 鉴权 + 数据范围校验后 Feign 调 `passenger` 对应内部接口。

---

### 4.6 角色 / 菜单维护（建议仅 SUPER）

首期可 **只通过 SQL 维护** `sys_menu`、`sys_role_menu`；若要做界面，可再增加：

- `GET/POST/PUT/DELETE /admin/api/v1/system/menus`
- `GET/PUT /admin/api/v1/system/roles/{id}/menus`

本文档不展开字段，避免与首期范围冲突。

---

### 4.7 业务数据域：省 / 市裁剪（`admin-api`）

与《权限清单与鉴权设计》**§6** 一致；实现类 **`AdminDataScope`**。非 **SUPER** 时：列表将请求中的 `provinceCode`/`cityCode` 与 JWT 域合并，**越界筛选用 403**；详情/删改目标不在域内用 **404**（不区分不存在与无权限）。**市隶属省**：`cityCode` 前缀与账号 `provinceCode` **前 2 位** 一致（与操作员建档等相同约定）。

**BFF 已接入**：订单；`/pricing/fare-rules`；`/capacity` 下公司 / 司机 / 司机车辆；`/capacity/team-change-requests`（含 `pending-count`）。**`capacity-service`**：司机分页与换队分页可接 `provinceCode`/`cityCode`；换队 VO 含 `driverCityCode`；`GET /api/v1/drivers/{id}` 供 BFF 校验司机域（直连暴露须另行加固）。

---

## 5. `passenger` 对内接口（仅 `admin-api` Feign）

**Base path 建议**：`/api/v1/admin/sys`（与 `/api/v1/customers` 并列，便于网关后期按路径隔离）。

**安全（首期）**：内网部署；**二期**可要求请求头 `X-Internal-Token` 与 `admin-api` 配置一致（见设计文档「后续演进」）。

以下均为 **JSON**，返回 **`ResponseVo`**。

---

### 5.1 凭据校验（登录第一步）

| 项 | 说明 |
|----|------|
| **POST** | `/api/v1/admin/sys/auth/verify-credentials` |

**请求体**：`username`，`password`。

**成功 `data`**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `userId` | long | `sys_user.id` |
| `tokenVersion` | long | 当前 `token_version` |
| `status` | int | 用户状态，非 1 则 admin-api 拒绝登录 |

**失败**：业务码非 200 或 404 风格（统一由 `admin-api` 转成「用户名或密码错误」）。

**实现**：BCrypt 比对 `password_hash`；勿返回「用户不存在」的区分文案。

---

### 5.2 安全上下文（JWT 校验 / Redis 回源）

| 项 | 说明 |
|----|------|
| **GET** | `/api/v1/admin/sys/users/{userId}/security-context` |

**成功 `data`**（供拼装 JWT 与 Redis）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `userId` | long | |
| `username` | string | |
| `displayName` | string \| null | |
| `provinceCode` | string \| null | |
| `cityCode` | string \| null | |
| `tokenVersion` | long | |
| `status` | int | |
| `roleCodes` | string[] | 如 `["PROVINCE_ADMIN"]` |
| `menuIds` | long[] | 可选；或仅由 5.3 树接口提供 |

**用户不存在或已删**：404。

---

### 5.3 用户菜单树

| 项 | 说明 |
|----|------|
| **GET** | `/api/v1/admin/sys/users/{userId}/menus` |

**成功 `data`**：与 **4.3** 相同结构的菜单树（服务端按 `userId` 关联角色与 `sys_role_menu` 查询）。

---

### 5.4 修改密码并递增 token_version

| 项 | 说明 |
|----|------|
| **PUT** | `/api/v1/admin/sys/users/{userId}/password` |

**请求体**：`oldPassword`，`newPassword`（或由 admin-api 在重置场景只传 `newPassword` + 内部凭证）。

**成功**：更新 `password_hash`，`token_version = token_version + 1`。

---

### 5.5 操作员 CRUD（供 admin-api 委托）

| Method | Path | 说明 |
|--------|------|------|
| `GET` | `/api/v1/admin/sys/operators` | Query：`provinceCode`（必填，由 BFF 从当前用户注入），`cityCode`，`pageNo`，`pageSize`，`username` |
| `POST` | `/api/v1/admin/sys/operators` | 创建 `CITY_OPERATOR`，body 含 `username`，`password`，`displayName`，`provinceCode`，`cityCode` |
| `GET` | `/api/v1/admin/sys/operators/{id}` | 详情 |
| `PUT` | `/api/v1/admin/sys/operators/{id}` | 更新 |
| `DELETE` | `/api/v1/admin/sys/operators/{id}` | 软删；**递增** `token_version` 若需立即踢下线 |

**说明**：**省/市与角色一致性**、**省管只能操作本省城市** 在 **passenger 服务内再次校验**（防绕过 BFF）。

---

## 6. JWT 载荷（`admin-api` 签发）

| Claim | 说明 |
|--------|------|
| `sub` | `sys_user.id`（字符串或数字，团队统一） |
| `tv` | `token_version`，与库表一致 |
| `exp` / `iat` | 标准时间 |
| 可选 `uname` | `username` |

校验：签名 + `exp` + **`tv` 与 5.2 或 Redis 中一致**。

---

## 7. Redis 键（建议）

| Key | 值 | TTL |
|-----|-----|-----|
| `adm:user:{userId}` | JSON：roleCodes、provinceCode、cityCode、menuIds 或树摘要、tokenVersion | 可配置，如 30～120 分钟 |

**失效**：用户 **角色/菜单/省市区/密码/status/token_version** 变更时 **删除该 key**。

---

## 8. 错误码与 HTTP 状态（摘要）

| 场景 | HTTP | 业务 code（若用 ResponseVo） |
|------|------|------------------------------|
| 未登录 / token 无效 / tv 不匹配 | 401 | 401 |
| 无菜单或无接口权限 | 403 | 403 |
| 登录失败（模糊） | 401 | 401 |
| passenger 不可用 | 502 | 502 |
